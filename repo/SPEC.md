Develop a Retail Operations & Customer Care Hub system for an on-premise specialty retailer that needs end-to-end ordering, pickup, and after-sales handling without any internet dependency. Customers use an Angular web app (English UI) on in-store kiosks or local Wi‑Fi to browse a locally hosted catalog with full-text search, multi-criteria filtering (price, availability, brand), and sorting; a configurable recommendation panel lets an administrator toggle ranking strategies and run A/B variants by store or date range, showing clear UI feedback such as "Test A active for 03/25/2026–04/08/2026." Shoppers place orders with combined payment recorded as tender splits (for example, $20.00 cash plus a "card terminal receipt reference" entered by staff) and can split items into separate pickup groups when inventory is staged in different departments. A 6-digit pickup code is generated at checkout, displayed on-screen, and can be printed; store associates verify it at handoff with a maximum of 5 attempts before requiring manager override. Customers can submit reviews with text, up to 6 images (JPG/PNG, each ≤ 5 MB), and one follow-up review within 14 days; they can also open after-sales tickets for returns, refunds, or price adjustments, track status, and receive outcome notifications inside the app (no email/SMS). Associates and supervisors work an "exam workflow" style console: check-in of returns at the counter, guided triage questions that route tasks across departments (front desk, fulfillment, accounting), interruption/retest handling when a return needs re-inspection, and a timeline view that shows each node's duration and current owner.

The backend uses Fastify to implement a decoupled REST API consumed by the Angular frontend over a local network, with PostgreSQL as the system of record for orders, tickets, reviews, campaigns, and audit trails. Order lifecycle rules are enforced server-side: carts reserve stock for 30 minutes before auto-cancel; out-of-stock triggers reassignment to a different pickup group or cancellation with a mandatory reason code; refunds/returns default to a 30-day window (manager can extend to 60 days), and price adjustments require an attached receipt reference and are capped at $50.00 per order unless the customer is in a top tier. A built-in rules engine drives decision orchestration for coupons, points/tier benefits, moderation, and risk limits using versioned rule sets with conditional expressions, allowlists/denylists, thresholds, grouping, serial/parallel evaluation, priority, and human-readable hit explanations; any rule change requires an admin comment and supports one-click rollback to the previous published version. Sensitive-content moderation runs fully offline using configurable banned-term dictionaries, simple pattern rules, and file validation (MIME/type checks plus SHA-256 hashing to detect re-uploaded flagged images); reported content enters an appeals queue with anti-fraud throttles (e.g., max 5 reports per user per day). Security is local-first: username/password authentication only (minimum 10 characters, lockout for 15 minutes after 5 failed attempts), role-based access control, encrypted secrets and sensitive notes at rest, and masking of customer phone numbers in staff views. All actions write immutable audit logs with timestamps, actor, before/after values, and node duration metrics to support reconciliation and operational accountability entirely offline.
